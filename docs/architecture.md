# Architecture

## Goal
Document the target architecture for the Hotel Finance Reconciliation System.

## Style
- extensible modular monolith
- clear separation between import, extraction, normalization, matching, exceptions, and reporting
- shared normalized model
- explainable and auditable reconciliation

## Initial module boundaries
- import
- extraction
- normalization
- matching
- exceptions
- reporting
- domain
- shared

## Notes
This file is a living architecture reference and should evolve with implementation.
